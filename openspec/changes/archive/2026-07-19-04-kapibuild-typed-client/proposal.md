## Why

Users need a type-safe way to perform CRUD operations on their API resources. The raw kapi-client works with StoredObject (JSON values), but users want to work with their typed wrapper structs (Widget, Gadget, etc.). A typed client provides generic CRUD operations that handle serialization/deserialization automatically.

## What Changes

- **Typed client library** — generic CRUD trait over kapi-client
  - Provides create, get, update, delete, list operations
  - Works with wrapper structs generated by kapi-derive
  - Handles serialization of spec/status fields
  - Handles deserialization of StoredObject back to typed structs

## Capabilities

### New Capabilities
- `04-kapibuild-typed-client`: Type-safe CRUD operations for kapi resources

### Modified Capabilities
(none)

## Impact

- **New library**: typed client module/crate
- **Dependencies**: Requires kapi-client, kapi-core, kapi-derive
- **Workflow**: Users use typed client in their controllers instead of raw kapi-client

## Non-goals

- Controller scaffolding (handled by `kapibuild controller generate`)
- Watch/streaming operations (future work)
- Schema registration

## Future Work
