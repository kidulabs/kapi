## ADDED Requirements

### Requirement: Typed client library
The system SHALL provide a typed client library that wraps kapi-client and provides generic CRUD operations on wrapper structs generated by kapi-derive.

#### Scenario: Use typed client for CRUD operations
- **WHEN** user creates a TypedClient<Widget>
- **THEN** user can call create, get, update, delete, list operations with Widget type

### Requirement: Generic CRUD trait
The system SHALL provide a generic TypedClient<T> that wraps KapiClient and provides typed operations for any wrapper struct T that implements the required traits.

#### Scenario: Create operation
- **WHEN** user calls typed_client.create(namespace, &resource) with a typed resource
- **THEN** system serializes spec to JSON, creates StoredObject via kapi-client, returns typed Widget

#### Scenario: Get operation
- **WHEN** user calls typed_client.get(namespace, name)
- **THEN** system fetches StoredObject via kapi-client, deserializes spec/status to typed fields, returns Widget

#### Scenario: Update operation
- **WHEN** user calls typed_client.update(widget) with a Widget
- **THEN** system serializes typed fields to StoredObject, updates via kapi-client, returns updated Widget

#### Scenario: Delete operation
- **WHEN** user calls typed_client.delete(namespace, name)
- **THEN** system deletes via kapi-client, returns deleted Widget

#### Scenario: List operation
- **WHEN** user calls typed_client.list(namespace, opts)
- **THEN** system lists via kapi-client, deserializes each item, returns Vec<Widget>

### Requirement: Serialization strategy
The system SHALL use serde for serialization. Spec/status fields SHALL be serialized as JSON values within StoredObject.

#### Scenario: Spec serialization
- **WHEN** a Widget with WidgetSpec is created
- **THEN** WidgetSpec is serialized to JSON and stored in StoredObject.spec field

#### Scenario: Status serialization
- **WHEN** a Widget with WidgetStatus is updated
- **THEN** WidgetStatus is serialized to JSON and stored in StoredObject.status field

### Requirement: Error handling
The system SHALL return Result<TypedError> for all operations. TypedError SHALL wrap ClientError.

#### Scenario: Error wrapping
- **WHEN** kapi-client returns an error
- **THEN** typed client wraps it in TypedError and returns it
