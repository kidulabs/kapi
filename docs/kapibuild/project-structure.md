# Project Structure

The `kapibuild init` command scaffolds a new kapi controller project with the following directory layout:

```
<project-name>/
├── Cargo.toml          # Project manifest with kapi dependencies
├── Kapifile            # YAML manifest tracking resources and domain
├── src/api/             # API type definitions (generated by `kapibuild api create`)
├── schemas/            # JSON schemas (generated by `kapibuild api generate`)
└── src/
    └── main.rs         # Entry point with Manager setup
```

## File Descriptions

### Cargo.toml

Contains project metadata and dependencies:
- **kapi-core**: Core types (StoredObject, WatchEvent, ResourceKey)
- **kapi-client**: HTTP client for communicating with kapi servers
- **kapi-controller**: Controller runtime SDK (Manager, Reconciler traits)
- **serde**: Serialization/deserialization with derive support
- **tokio**: Async runtime
- **tracing**: Structured logging
- **async-trait**: Async trait support
- **schemars**: JSON Schema generation from Rust types

All kapi dependencies are pulled from the workspace via path dependencies.

### Kapifile

YAML manifest tracking project metadata:
```yaml
domain: kapi.io
version: v1
resources: []
```

- **domain**: API group for resources (e.g., `kapi.io`)
- **version**: Default API version (e.g., `v1`)
- **resources**: List of resources managed by this project

### src/api/

Directory for API type definitions. Use `kapibuild api create` to generate types here.

### schemas/

Directory for generated JSON schemas. Use `kapibuild api generate` to compile types into schemas.

### src/main.rs

Entry point with Manager initialization:
```rust
use kapi_controller::Manager;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    tracing_subscriber::fmt::init();
    
    let manager = Manager::builder()
        .build()
        .await?;
    
    // TODO: Register controllers here
    
    manager.start().await?;
    
    Ok(())
}
```

## Usage

```bash
# Initialize in current directory
kapibuild init my-controller

# Initialize at specific path
kapibuild init /path/to/my-controller
```

The generated project uses kapi dependencies from crates.io and is fully standalone.
